# IAM Identity Boundary Platform

## Keynote

This project shows how to keep AWS identity under control. It includes permission boundaries, OIDC-based automation access, and a least-privilege role model that looks like real cloud security work.

## Best for

- IAM engineer
- Cloud security engineer
- DevOps engineer owning access controls

- AWS STS
- GitHub OIDC
- AWS Organizations concepts
- Terraform

## What it proves

- Permission boundary design
- Federated CI/CD access
- Least privilege by default
- Reusable identity baselines
